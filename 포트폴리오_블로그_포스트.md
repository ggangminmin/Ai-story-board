## 🎯 이 프로젝트가 뭔가요?

ChatGPT처럼 AI가 자동으로 메모를 요약해주는 메모장 웹사이트를 만들었어요.

---

## 🌟 뭘 할 수 있나요?

1. **메모 작성** - 글을 쓰면 AI가 자동으로 요약해줘요
2. **파일 업로드** - PDF, 워드, 엑셀, 이미지 첨부 가능
3. **스마트 검색** - "여행"이라고 검색하면 관련 메모를 다 찾아줘요

6. **앱처럼 사용** - 핸드폰 홈 화면에 추가 가능

## 🔧 겪었던 문제와 해결

### 문제 1: "배포하니까 안 돼요!"
**증상**: 내 컴퓨터에서는 잘 됐는데 인터넷에 올리니까 작동 안 함

**원인**: 내 컴퓨터 주소(localhost)를 그대로 써서

**해결**: 배포된 서버 주소로 변경
```javascript
// 이전: localhost:3001 (내 컴퓨터)
// 이후: ai-story-board.onrender.com (인터넷 서버)
```

---

### 문제 2: "저장이 안 돼요!"
**증상**: 메모를 저장하려고 하면 에러 발생

**원인**: 보안 설정(CORS) 때문에 차단됨

**해결**: 서버에서 내 웹사이트를 허용 목록에 추가
```javascript
// "이 웹사이트들은 내 서버 사용해도 돼!" 라고 설정
허용 목록: ai-story-board-sable.vercel.app
```

---

### 문제 3: "빌드가 실패했어요!"
**증상**: Vercel/Render에서 자동으로 설치할 때 에러

**원인**: TypeScript 관련 패키지가 잘못된 곳에 있었음

**해결**: 필요한 패키지를 올바른 위치로 이동
```
devDependencies (개발용) → dependencies (실제 사용)
```

## 🎓 이 프로젝트에서 배운 것

### 1. 프론트엔드 vs 백엔드
**쉽게 설명하면**:
- **프론트엔드** = 식당 홀 (손님이 보는 메뉴판, 테이블)
- **백엔드** = 식당 주방 (요리하고 재료 보관하는 곳)

**왜 나눴을까요?**
- 보안: 비밀 정보(API 키)를 손님한테 안 보여주기 위해
- 역할 분담: 화면은 화면대로, 데이터는 데이터대로

### 2. 배포가 뭔가요?
개발 = 내 컴퓨터에서만 작동
배포 = 인터넷에 올려서 누구나 접속 가능

**비유하면**:
- 개발 = 집에서 혼자 요리 연습
- 배포 = 실제 식당 오픈

### 3. 환경변수가 뭔가요?
비밀번호나 API 키 같은 민감한 정보를 코드에 직접 쓰지 않고 따로 저장하는 방법

**나쁜 예**:
```javascript
const apiKey = "sk-1234567890abcdef"  // ❌ 코드에 직접
```

**좋은 예**:
```javascript
const apiKey = process.env.OPENAI_API_KEY  // ✅ 따로 저장
```
